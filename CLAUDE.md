# 応用情報技術者試験 学習リポジトリ - Claude Code 設定

## プロジェクト概要
応用情報技術者試験（2026年11月 前期）の合格を目指す学習管理リポジトリ。

## 受験者プロフィール（個人情報は記載しない）
- 開発エンジニア 2年
- クラウドエンジニア 1年
- 保有資格：AWS CLF / AIF / SAA / SOA / DVA / 基本情報技術者
- 目標：午前・午後ともに75点以上
- 学習期間：2026年5月〜11月（約6ヶ月・週9時間ペース）

## コマンド実行の許可要求ルール
コマンド実行の許可を求める際は、以下のルールに従う。
- **必ず日本語で説明する**
- 何のコマンドを実行しようとしているか
- なぜそのコマンドが必要か（目的・理由）
- 実行するとどうなるか（影響・副作用）
を明記してから許可を求める。

## ⚠️ 個人情報ポリシー（必ず遵守）
- 氏名・企業名・所属・連絡先などの個人を特定できる情報は一切記載しない
- パスワード・APIキー・トークン等の認証情報は絶対に記載しない
- 固有名詞（職場名・学校名・プロジェクト名等）は記載しない
- スコアや日付などの学習記録は記載してよい

## 問題レビューフロー（毎日の学習）

### 手順
1. fe-siken.com または ap-siken.com で問題を解く
2. 解説を読んだあと、以下をまとめてClaudeに貼り付ける

```
【問題文】
（全文貼り付け）

【解説】
（全文貼り付け）

【結果】正解 or 不正解
【難易度】難 or 中 or 易
```

3. Claudeが自動で以下を行う：
   - カテゴリ判定
   - `01_morning/[カテゴリ]/summary.md` に復習サマリーを追記
   - 不正解の場合は `03_weak_points/morning_weak.md` にも追記

### サマリーのフォーマット（復習時にパッと分かる構成）
```
## [トピック名]｜★難/中/易｜✅正解 or ❌不正解｜日付

### 概要
何を問う問題か（1〜2文）

### ポイント
- 重要な概念・定義（3〜5個）

### 要点（これだけ覚える）
> 試験で即答するために必要な核心（1〜2行）

### ひっかかりやすい罠
- 間違いやすいポイント
```

### 使用サイトと位置づけ
| サイト | 対象 | 使い方 |
|--------|------|--------|
| fe-siken.com | 基本情報（FE） | 5〜6月の基礎固め期に使用 |
| ap-siken.com | 応用情報（AP） | 6月以降はこちらをメインに移行 |
