# スキル：まとめ生成（generate_summary）

## 概要
カテゴリ別のまとめmdをコンテンツ作成エージェントが生成する際のルール。

## 使用方法
```
「コンテンツ作成エージェントとして、
  01_morning/06_networks/ にTCP/IPのまとめを生成して」
```

## 生成ルール
1. ファイル名はトピックを英語スネークケースで命名（例：`tcp_ip.md`）
2. 重要度・出題頻度を冒頭に記載
3. AWSとの対比欄を必ず含める
4. 表・箇条書き中心で長文段落は避ける
5. 個人情報（氏名・企業名・固有名詞）は含めない
6. 最終行に `最終更新：YYYY-MM-DD` を記載

## カテゴリ対応表
| フォルダ | 対象トピック例 |
|---------|-------------|
| 01_morning/06_networks | TCP/IP, DNS, HTTP, ルーティング |
| 01_morning/07_databases | SQL, 正規化, トランザクション |
| 01_morning/08_information_security | 暗号, 認証, 攻撃手法, PKI |
| 02_afternoon/01_security_required | 過去問解説, 頻出パターン |

# スキル：模試採点・スコア記録（score_exam）

## 概要
過去問・模試のスコアを記録・分析するスキル。

## 使用方法
```
「模試スコアを記録して。
  実施日：2026-06-15
  対象：令和5年秋期 過去問
  午前：68点
  午後：72点（セキュリティ16点、NW18点、DB14点、PM12点、サービスMG12点）」
```

## 記録先
`05_progress/mock_exam_results.md` に追記

## 記録フォーマット
```markdown
| YYYY-MM-DD | 令和X年 春/秋期 | XX点 | XX点 | [メモ] |
```

## 分析時の出力フォーマット
```markdown
## スコア分析（YYYY-MM-DD時点）

### 現状
- 午前平均：XX点（目標：75点、差：XX点）
- 午後平均：XX点（目標：75点、差：XX点）

### カテゴリ別弱点
1. [最も改善が必要なカテゴリ]：XX点
2. [次に改善が必要なカテゴリ]：XX点

### 戦略アドバイス
- [具体的なアクション]
```

## 禁止事項
- 個人を特定できる情報はスコア記録に含めない
